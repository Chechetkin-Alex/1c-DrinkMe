# Бизнес-ценность

## Идея

DrinkMe -- сервис для заказа кофе, выпечки и товаров для кофе онлайн заранее. 

Мотивация: сделать локальный аналог Drinkit, но для жителей Долгопы. Сейчас, насколько я знаю, их там нет, а сценарий довольно интуитивный: студент/житель долгопы заранее выбирает кофе, булочку или пачку зерна, оплачивает, а потом просто забирает его без очереди. То есть модель Заказал, оплатил, дошел до магазина, забрал.

## Проблема

В районе МФТИ есть спрос на быстрые заказы еды и кофе на вынос (верим в предпосылку).

Обычный сайт кофейни часто просто показывает меню. В этом проекте я хочу сделать именно сервис, где пользователь может собрать заказ и оформить его заранее.

Киллер фичей станет студенческое комбо, субсидированное добрым дядей инвестором, выпускником физтеха. Оно будет доступно, если клиент в личном кабинете введет почту в формате "@phystech.edu".

## Целевая аудитория

Основная аудитория:

- студенты, сотрудники МФТИ
- жители долгопы

Также есть отдельная роль администратора кофейни. Он управляет товарами, категориями, остатками и статусами заказов.

## Основные сценарии

Пользователь может:

- посмотреть каталог напитков, выпечки и товаров
- открыть карточку товара
- добавить товары в корзину
- оформить заказ на вынос
- посмотреть историю своих заказов
- повторить похожий заказ в будущем

Администратор может:

- добавлять и редактировать товары
- менять остатки
- смотреть заказы
- менять статус заказа

## Первая версия

В первую версию входят:

- каталог напитков, выпечки, зерна и аксессуаров
- категории товаров
- карточка товара
- корзина
- оформление заказа
- регистрация и вход
- история заказов пользователя
- статусы заказа
- регистрация в домене @phystech.edu => студенческое комбо с меньшей ценой
- админка для управления товарами и заказами

## Планируемый стек

- Бэкенд: Python, Django, Django REST Framework
- Авторизация: токены Django REST Framework
- PostgreSQL для финального запуска
- Фронтенд: React, Vite, React Router
- Управление состоянием: Context API или Zustand
- Тесты: pytest, pytest-django
- Инструменты: Docker Compose, ruff/black по возможности

## Основные модули

- `users` - пользователи, регистрация, профиль, права доступа
- `catalog` - категории, напитки, товары, остатки, цены
- `cart` - корзина пользователя
- `orders` - оформление заказов, позиции заказа, статусы
- `reviews` - отзывы к товарам

## Локальный запуск бэкенда

Создать окружение:

```bash
python3 -m venv .venv
```

Установить зависимости:

```bash
.venv/bin/python -m pip install -r backend/requirements.txt
```

Проверить проект:

```bash
cd backend
../.venv/bin/python manage.py check
```

Применить миграции:

```bash
cd backend
../.venv/bin/python manage.py migrate
```

Запустить сервер:

```bash
cd backend
../.venv/bin/python manage.py runserver
```

Наполнить базу демо-данными:

```bash
cd backend
../.venv/bin/python manage.py seed_demo
```

После этого в админку можно войти с логином `admin` и паролем `222333`.
