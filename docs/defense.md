# Подготовка к защите

## Короткое описание

DrinkMe это учебный сервис предзаказа кофе, выпечки и товаров для кофе рядом с МФТИ. Пользователь может посмотреть каталог, выбрать товар, добавить его в корзину, оформить заказ и оставить отзыв. Администратор управляет каталогом, остатками, заказами и отзывами через Django admin.

## Что закрывает проект

- Django и Django REST Framework
- React и Vite
- REST API
- база данных и связи между моделями
- авторизация по токену
- права доступа
- тесты
- сервисный слой для оформления заказа
- админка Django
- демо-данные через management command
- минимальные настройки для прод-запуска

## REST API

В проекте есть ресурсы:

- `/api/categories/`
- `/api/products/`
- `/api/cart/`
- `/api/orders/`
- `/api/reviews/`
- `/api/auth/`

Методы используются по смыслу:

- `GET` получает данные
- `POST` создает данные
- `PATCH` частично обновляет данные
- `DELETE` удаляет данные

## База данных

Основные модели:

- пользователь
- категория
- товар
- корзина
- позиция корзины
- заказ
- позиция заказа
- отзыв

Важная деталь: в позицию заказа копируется название и цена товара. Это нужно, чтобы старые заказы не менялись после изменения товара в каталоге.

## Авторизация

Используются токены Django REST Framework. После входа или регистрации сервер возвращает токен. Фронтенд хранит его в `localStorage` и отправляет в заголовке:

```text
Authorization: Token <token>
```

Так бэкенд понимает, чей профиль, корзина и заказы обрабатываются.

## Права доступа

- каталог можно смотреть без входа
- корзина доступна только авторизованному пользователю
- заказы доступны только авторизованному пользователю
- пользователь видит только свои заказы
- отзывы можно смотреть без входа
- отзыв можно создать только после входа
- менять и удалять можно только свой отзыв
- администратор управляет каталогом и заказами

## Сервисный слой

Оформление заказа вынесено в `orders/services.py`.

Функция `create_order_from_cart` делает несколько действий:

1. проверяет, что корзина не пустая
2. проверяет доступность товаров
3. проверяет остатки
4. создает заказ
5. создает позиции заказа
6. уменьшает остатки товаров
7. очищает корзину

Это сделано, чтобы сложная бизнес-логика не лежала прямо во view.

## Тесты

Проверяются основные сценарии:

- список товаров доступен без входа
- фильтрация товаров работает
- гость не может менять каталог
- регистрация и вход работают
- текущий пользователь возвращается по токену
- корзина требует авторизацию
- товар добавляется в корзину
- нельзя добавить больше остатка
- заказ создается из корзины
- после заказа уменьшается остаток
- пользователь видит только свои заказы
- отзывы доступны для просмотра
- гость не может создать отзыв
- пользователь не может редактировать чужой отзыв
- демо-данные создаются командой

Запуск:

```bash
cd backend
../.venv/bin/python -m pytest
```

## Почему не сложнее

Проект учебный, поэтому часть вещей сделана осознанно просто:

- нет реальной оплаты
- нет доставки
- нет сложной системы скидок
- нет отдельной панели администратора на React
- нет промышленной настройки деплоя

При этом проект покрывает основной стек курса и показывает законченный пользовательский сценарий.

## Прод-запуск

Для простого прод-запуска добавлен `Procfile` и зависимость `gunicorn`.

Через переменные окружения можно задать:

- секретный ключ Django
- режим отладки
- список разрешенных хостов
- HTTPS-редирект
- secure cookies
- HSTS
- HSTS для поддоменов и preload при необходимости

Локально проект по умолчанию продолжает работать с SQLite и `DEBUG=True`.
